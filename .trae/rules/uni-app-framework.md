# 项目技术框架规则

本项目所有功能模块的开发、设计与实现必须严格基于 uni-app 技术框架。

## 适用范围

此规则适用于以下所有开发环节：

- 功能需求分析
- 技术方案设计
- 代码编写
- 架构规划
- API 设计与封装
- 跨平台兼容性处理

## 核心要求

1. **严格基于 uni-app 技术体系**：所有功能实现必须依赖 uni-app 提供的 API、生命周期、组件模型和页面路由机制，不得绕过或偏离 uni-app 技术体系。

2. **不得引入平台特定替代方案**：禁止使用绕过 uni-app 的平台原生 API 或第三方方案来替代 uni-app 已有能力（如使用原生路由替代 uni-app 路由、使用原生存储替代 uni.storage 等）。

3. **API 能力边界以 uni-app 官方文档为准**：当需要使用某项能力时，优先查阅 uni-app 官方文档确认其支持情况。uni-app 明确不支持的特性，不得假设其可用或尝试绕过限制传递参数。

4. **跨平台一致性优先**：设计和实现时需考虑 uni-app 所支持的各平台（App、H5、小程序）的行为差异，不得仅针对单一平台做假设。
